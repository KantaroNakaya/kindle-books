## 第 1 章：LookerStudio とは何か？

### 1-1. LookerStudio って何？

**GoogleAnalytics と SearchConsole のデータを自動レポート化するツール**

LookerStudio（ルッカー・スタジオ）は、Google が提供する無料のデータビジュアライゼーションツールです。

![LookerStudioのホーム画面](assets/images/chapter1/lookerstudio-home.png)
_図 1-1: LookerStudio のホーム画面。直感的なインターフェースで誰でも簡単にレポートを作成できます。_

**🎯 本書で扱う内容**

-   GoogleAnalytics（GA4）のデータを自動でレポート化
-   GoogleSearchConsole のデータを自動でレポート化
-   両方のデータを組み合わせた統合レポートの作成

**🔍 具体的に何ができるの？**

**GoogleAnalytics データの活用：**

-   サイトのアクセス数を棒グラフで表示
-   ユーザーの行動フローを可視化
-   コンバージョン率のトレンド分析
-   デバイス別・地域別のアクセス分析

![GA4データの可視化例](assets/images/chapter1/ga4-visualization.png)
_図 1-2: GoogleAnalytics のデータを美しいグラフに変換した例。アクセス数の推移が一目で分かります。_

**SearchConsole データの活用：**

-   検索順位の推移を折れ線グラフで表示
-   クリック数とインプレッション数の関係
-   検索キーワード別のパフォーマンス
-   ページ別の検索流入分析

![SearchConsoleデータの可視化例](assets/images/chapter1/searchconsole-visualization.png)
_図 1-3: SearchConsole のデータを可視化した例。検索順位とクリック数の関係が明確に分かります。_

**💡 なぜ「自動レポート化」が重要なのか？**

GoogleAnalytics でもグラフや表は見られますが、毎回手動で確認するのは大変です。

**GA4 の限界：**

-   毎回ログインして確認が必要
-   複数のレポートを同時に見られない
-   チームメンバーへの共有が面倒
-   カスタマイズの自由度が低い

**LookerStudio の強み：**

-   データが自動で更新される
-   複数の指標を一画面で確認
-   URL 一つでチーム全員に共有
-   自由自在にカスタマイズ可能

**🎯 GA4 でもグラフが見られるのに、なぜ LookerStudio を使うのか？**

**1. 時間の節約**

-   GA4：毎回ログイン → レポート選択 → 期間設定 → 確認
-   LookerStudio：URL を開くだけで最新データを確認

-   LookerStudio：URL 一つでリアルタイム共有

### 1-2. なぜ今 LookerStudio なのか？

**サイト解析の自動化がビジネス成功の鍵**

現代の Web マーケティングでは、データに基づいた迅速な意思決定が求められています。

**📊 サイト解析の重要性**

-   ユーザーの行動パターンを理解
-   SEO 施策の効果測定
-   コンテンツマーケティングの最適化
-   競合他社との差別化

**⚡ 従来のサイト解析の問題点**

**GoogleAnalytics だけを使う場合：**

1. **時間がかかる**：毎回ログインして複数のレポートを確認
2. **見落としが多い**：重要な指標を見逃すリスク
3. **共有が困難**：チームメンバー全員に GA4 の権限が必要
4. **統合分析ができない**：Analytics と SearchConsole のデータを同時に見られない

<!-- 画像: GA4の複数レポート確認の手間を示すスクリーンショット -->
<!-- 説明: GA4で複数のレポートを確認する際の手間を視覚化 -->

**Excel で手作業する場合：**

1. **手間が膨大**：データをエクスポート → 加工 → グラフ作成
2. **ミスが多い**：手作業によるデータ入力ミス
3. **更新が面倒**：毎回同じ作業を繰り返す必要
4. **リアルタイム性がない**：過去のデータで分析

-   手作業によるミスを完全に排除

### 1-3. 初心者が知っておくべき基本用語

**サイト解析に特化した用語を理解しよう**

LookerStudio でサイト解析を行うために、必要な用語を押さえておきましょう。

**📊 データソース（Data Source）**

データの元となる場所のことです。

**本書で使用するデータソース：**

-   GoogleAnalytics（GA4）
-   GoogleSearchConsole
-   Google スプレッドシート（補助的に使用）

<!-- 画像: データソース設定画面 -->
<!-- 説明: LookerStudioでのデータソース設定画面 -->

**💡 ポイント：**
データソースを設定することで、LookerStudio がそのデータを読み込めるようになります。

**📈 レポート（Report）**

グラフやチャートを配置するページのことです。

**サイト解析での活用：**

-   ページ 1：サイト全体の概要
-   ページ 2：SEO 分析
-   ページ 3：ユーザー行動分析
-   ページ 4：コンバージョン分析

<!-- 画像: マルチページレポートの例 -->
<!-- 説明: 複数ページで構成されたレポートの例 -->

**🎛️ ダッシュボード（Dashboard）**

複数のグラフを一画面にまとめた表示のことです。

**サイト解析でのメリット：**

-   重要な指標を一目で確認
-   関連するデータをまとめて表示
-   意思決定のスピードアップ

<!-- 画像: ダッシュボードの例 -->

**🔧 ウィジェット（Widget）**

レポートに配置するグラフやテキストなどの要素のことです。

**サイト解析でよく使う種類：**

-   時系列グラフ（アクセス数・検索順位の推移）
-   棒グラフ（ページ別・キーワード別の比較）
-   円グラフ（デバイス別・地域別の割合）
-   テーブル（詳細データの表示）
-   スコアカード（重要な KPI の表示）

**📋 フィールド（Field）**

データの列や項目のことです。

**GA4 の主要フィールド：**

-   日付
-   ページビュー数
-   ユーザー数
-   セッション数
-   コンバージョン数
-   地域
-   デバイス

**SearchConsole の主要フィールド：**

-   日付
-   検索クエリ
-   ページ URL
-   クリック数
-   インプレッション数
-   CTR（クリック率）
-   平均順位

データを特定の条件で絞り込む機能です。

**サイト解析での使い方：**

-   特定の期間のデータのみ表示
-   特定のページのデータのみ表示
-   特定の地域のデータのみ表示
-   特定のデバイスのデータのみ表示

**📊 メトリクス（Metrics）**

数値で表される指標のことです。

**GA4 の主要メトリクス：**

-   ページビュー数
-   ユーザー数
-   セッション数
-   コンバージョン数
-   直帰率
-   平均セッション時間

**SearchConsole の主要メトリクス：**

-   クリック数
-   インプレッション数
-   CTR（クリック率）
-   平均順位

**🏷️ ディメンション（Dimensions）**

データを分類する項目のことです。

**GA4 の主要ディメンション：**

-   日付
-   ページ URL
-   地域
-   デバイス
-   ユーザータイプ

**SearchConsole の主要ディメンション：**

-   日付
-   検索クエリ
-   ページ URL
-   国
-   デバイス

**💡 初心者向けの覚え方**

-   **メトリクス**：「何個」「いくら」などの数値（ページビュー数、クリック数など）
-   **ディメンション**：「いつ」「どこで」「何で」などの分類（日付、地域、デバイスなど）
